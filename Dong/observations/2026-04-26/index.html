1. Introducing GPT-5.5

内容简介

OpenAI 发布 GPT-5.5，定位为当前最强模型，重点强调 coding、研究、跨工具数据分析等复杂任务能力，速度同步提升，信号指向底座模型对多工具协同场景的原生支持。

关键要点

  • 不是渐进升级，是底座代际换代，coding 被列为一级能力而非附属
  • 官方强调「across tools」，意味着模型层原生多工具调度能力增强
  • 速度与能力同时提升，暗示推理效率架构有实质改进

对我们的启示

  • 应用层做的 prompt 链 / 工具链编排可能被模型层部分吸收，3-6 个月内需重新评估自有 agent 框架中哪些编排逻辑仍有价值
  • LLM 抽象层「可换 backend」的设计被再次验证为必要架构决策，而非过度工程
  • API 公开后应尽快将 GPT-5.5 加入候选 backend 做 A/B，尤其在多步工具调用场景下对比现有模型的 pass rate 和延迟
  • 若模型原生 tool-use 能力足够强，现有 ReAct / plan-and-execute 等框架的附加值会缩小，需提前想清楚自己的差异化在哪一层

来源：Introducing GPT-5.5

2. GPT-5.5 System Card

内容简介

OpenAI 发布 GPT-5.5 的 System Card，即该模型的安全与能力评估技术文档，涵盖红队测试结果、agentic 场景下的风险评估、各项 benchmark 表现及缓解措施，是主发布公告的补充性技术披露。

关键要点

  • System Card 是获取模型 agentic 能力边界与已知失败模式的一手数据源
  • 文档通常包含多步工具调用、自主执行等 agentic benchmark 的定量结果
  • 安全评估部分会披露模型在无人监督长链任务中的对齐与可控性测试

对我们的启示

  • 从 System Card 中提取 agentic benchmark 数据，可直接用于校准自有 agent 系统对 GPT-5.5 的能力预期，减少盲测成本
  • 关注文档中披露的工具调用失败模式和安全护栏触发条件，据此调整 agent 框架的错误处理与 fallback 策略
  • 对比 GPT-5.5 与前代 System Card 中 agentic 评估维度的变化，可判断 OpenAI 对「模型自主行动」的产品化方向和限制边界

来源：GPT-5.5 System Card

3. A pelican for GPT-5.5 via the semi-official Codex backdoor API

内容简介

GPT-5.5 已发布但官方 API 尚未开放，Simon Willison 通过第三方 agent 工具 OpenClaw 的订阅通道「后门」接口访问 GPT-5.5 并运行基准测试，同时讨论了 agent 工具绕过官方 API、借助订阅机制调用模型这一灰色地带的持续争议。

关键要点

  • GPT-5.5 面向 ChatGPT 付费用户和 Codex 开放，但 API 以安全为由延迟发布
  • 第三方 agent 工具（OpenClaw 等）通过订阅通道而非官方 API 接入新模型，形成事实上的非官方接口
  • 模型厂商与 agent 工具在接口访问权限上的博弈已成为持续性议题

对我们的启示

  • 依赖非官方或灰色通道接入模型的 agent 系统面临随时被切断的风险，架构设计需预留 fallback 路径和多 backend 切换能力
  • 模型厂商对 API 发布节奏的主动控制，意味着 agent 系统的模型升级窗口不由自己决定，产品路线图需考虑这一延迟
  • 订阅通道与 API 通道的权限分化可能催生新的接入合规标准，做 agent 基础设施的团队应关注 ToS 变化并评估合规成本

来源：A pelican for GPT-5.5 via the semi-official Codex backdoor API

5. DeepSeek V4 - almost on the frontier, a fraction of the price

内容简介

DeepSeek 发布 V4 系列预览版，包含 Pro（1.6T 总参数 / 49B 激活）和 Flash（284B / 13B 激活）两款 MoE 模型，均支持 100 万 token 上下文，MIT 开源；Pro 成为当前最大开放权重模型，定价远低于同级闭源模型。

关键要点

  • Pro 1.6T 参数刷新开放权重模型规模上限，激活参数仅 49B
  • Flash 13B 激活参数，有望在消费级硬件本地运行
  • 百万 token 上下文 + MIT 协议，降低长上下文 agent 部署门槛
  • 性能接近前沿但未专门强调 agentic 或 tool-use 能力

对我们的启示

  • Flash 的低激活参数量使本地 agent loop 的成本模型出现新选项，值得在延迟敏感场景做基准测试
  • 百万上下文窗口可减少 agent 多轮检索和分片策略的复杂度，但需实测长上下文下工具调用的准确率是否衰减
  • 开放权重 + MIT 协议意味着可以针对 tool-use / function-calling 做领域微调，团队如有私有部署需求可提前排期评估
  • 模型本身未突出 agentic 能力，短期内不会替代专门优化过 agent 行为的闭源模型，优先级可放在评估而非迁移

来源：DeepSeek V4 - almost on the frontier, a fraction of the price

6. llm 0.31

内容简介

Simon Willison 的命令行 LLM 工具发布 0.31 版，主要变更是接入 GPT-5.5、为 GPT-5 系列新增 verbosity 参数（控制输出详略）以及 image_detail 选项，同时修复了自定义 OpenAI 模型的异步注册问题。

关键要点

  • GPT-5.5 作为新 backend 被第三方 CLI 工具快速跟进
  • verbosity 参数是 OpenAI GPT-5 系列新增的模型级控制维度
  • image_detail 选项在 GPT-5.4/5.5 上多出「original」级别

对我们的启示

  • verbosity 这类模型侧参数正在把部分 prompt engineering 工作下沉为 API 选项，agent 框架的参数抽象层需要跟进适配
  • 如果你的 agent 管道用 llm CLI 做原型验证或脚本集成，0.31 可直接切 GPT-5.5 做对比测试，省去手动配置
  • image_detail 新增「original」级别意味着高保真视觉输入成本可能上升，多模态 agent 的 cost 控制策略需要把这个参数纳入考量

来源：llm 0.31

DeepSeek 发布 V4 系列两款 MoE 模型——V4 Pro（1.6T 总参 / 49B 激活）和 V4 Flash（284B 总参 / 13B 激活），均提供 Base 与 Instruct 版本，并明确支持华为昇腾芯片运行；但在主流 benchmark 上未能夺回榜首。

对我们的启示

  • 如果团队在评估自托管 backend，V4 Flash 的 13B 激活量意味着推理成本可能大幅低于同级闭源 API，值得跑一轮 agent 任务基准对比
  • 昇腾适配降低了在受限供应链环境下部署 agent 系统的硬件门槛，对有合规或地缘约束的团队是新选项

8. Quoting Romain Huet

内容简介

OpenAI Romain Huet 确认，从 GPT-5.4 起 Codex 已并入主模型，不再维护独立代码模型线；GPT-5.5 延续这一路线，重点强化 agentic coding 和 computer use，不会发布单独的 GPT-5.5-Codex 变体。

关键要点

  • Codex 代码模型线从 GPT-5.4 起正式合并进主模型，不再独立迭代
  • GPT-5.5 将 agentic coding 与 computer use 列为主线能力方向
  • OpenAI 多模型线（base / code）走向单一统一模型

对我们的启示

  • 依赖独立 Codex 端点做模型路由的 agent 系统需尽快迁移到统一模型 API，避免后续端点废弃带来的中断
  • 主模型原生覆盖强 coding 能力后，应用层「按任务类型选模型」的分发逻辑可简化，减少编排复杂度
  • coding 与非 coding 任务可在同一上下文窗口无缝切换，multi-step agent 中跨能力调度的 token 开销和延迟有望降低

来源：Quoting Romain Huet
