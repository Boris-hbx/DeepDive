# 产品定义：洞察探索（DeepDive）

> 一页纸的产品定义。先填粗，再补细。
>
> 状态：🟢 MVP 范围草稿（待团队评审）·  最近更新：2026-04-25

## 一句话

为团队成员持续追踪 **Agentic Software Engineering**（AI 时代的软件工程）领域的动态，自动提炼**带原文引用、可追溯**的洞察，让每个人每天用 5 分钟掌握当天值得关注的事。

差异点：可追溯（每条洞察可点回原文）、信息密度高（去重 + 诚实兜底）、为专业读者服务而非泛媒体读者。

## 目标用户

**MVP 阶段：只服务团队自己（8 人），团队共用一份 brief。**

理由：dogfooding 用户反馈最快，专业读者能容忍粗糙、能给精准反馈。等产品对自己人都好用了再外推。

后续扩展（不在 MVP 范围）：

- 公司领导 — 要更短的趋势判断与摘要
- 公司同事 — 要按主题搜索 / 订阅

## 核心场景

收敛为 3 个用户 job：

### Job 1 — 每日 brief（**MVP 唯一聚焦**）

团队成员早上打开 web，5 分钟扫完当日 Agentic Software Engineering 领域值得关注的事；每条带原文引用，可点回去深读。

### Job 2 — 主题深挖（P1，MVP 不做）

团队成员对某个主题（如 "agent 工程化"）、某个事件、某个公司（如 Anthropic 最近动作）发起追问，系统输出可交互探索 + 可导出的深度报告。

### Job 3 — 重大事件即时推送（P2，MVP 不做）

系统识别到重大事件（需谨慎判断，避免噪音）→ 通过邮件 / IM 即时推送给团队。

## 最小可用形态（MVP）

**单一形态：每日 brief**。目标：几周内团队自己每天用上。

- **数据来源**：人工指定 5–10 个高质量源（HN、固定 newsletter、RSS、Anthropic / OpenAI / Google 等官方 blog 等）。**不做自动发现**。具体清单待团队定。
- **主题范围**：单一主题——**Agentic Software Engineering**（agent 工程化、AI 时代软件研发范式、人机 / 多 agent 协作、相关工具链等）。Security 推到后续。
- **交互形态**：**静态网页站点**，按日期列出历史 brief，点进去看当天内容。**无登录、无后端服务**——最简单跑通即可。
- **输出**：单篇 Markdown brief，固定结构：
  - 「最关注的事」（≤3 条，每条 1 段话 + 原文链接）
  - 「值得一看的事」（≤5 条，每条 1 句话 + 原文链接）
  - 当日无重要事件时直接说"今日无重要事件"，**不凑数**
- **生成方式**：每天定时跑一次（也支持手动触发）。流水线：抓取 → 去重 → 摘要 → 生成 brief（输出 Markdown / HTML，发布到静态站点）。
- **平台**：Web（静态站点）。

## 不做什么（明确范围）

> 写下来不做的事，比写做的更重要。当前阶段刻意排除以下，为的是把单一形态做扎实。

- **暂不做**：用户登录 / 认证 — *团队内网共用，先把跑通最重要*
- **暂不做**：动态后端（数据库、API 服务）— *静态站点先跑通，brief 文件直接发布*
- **暂不做**：移动端原生 app — *先验证产品形态*
- **暂不做**：自动发现信息源 — *质量难保证，先用人工指定的源跑通链路*
- **暂不做**：多主题 / 多用户 / 订阅管理 — *MVP 单主题、团队共用*
- **暂不做**：邮件 / IM 推送 — *触达形态，brief 产出质量稳定后再加*
- **暂不做**：辣评 / 蓝军视角 / 风格化输出 — *先把"中性、可追溯"做扎实*
- **暂不做**：趋势预测 / 根因预判 — *容易不靠谱；先只梳理"已发生的"*
- **暂不做**：交互式深挖 / 深度案例报告 — *是另一种产品形态，不是 brief 的延伸*
- **暂不做**：Security 主题 — *推到 AI 软工主题跑顺之后*
- **暂不做**：用户反馈学习循环（赞 / 踩 / 个性化）— *数据量不够；先记日志为后续准备*

## 开放问题

> 还没想清楚的问题。**AI 看到本节非空时应停下来问，不要自行脑补。**

- **去重策略**：同一事件多家报道时怎么聚类？URL / 标题相似度 / 语义？
- **诚实兜底阈值**：什么算"今日无重要事件"？怎么量化判断？
- **静态站点的发布与运行**：brief 生成后发布到哪（GitHub Pages / 内网静态服务器 / 其他）？谁触发每日生成（本地脚本 / GitHub Action / cron）？
- **数据源清单的维护方式**：写死在代码 / 配置文件 / 待 P1 引入 UI？
