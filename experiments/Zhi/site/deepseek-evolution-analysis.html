三、DeepSeek 的优化思路与技术趋势

核心哲学：不追求暴力扩参数，而是在固定算力预算下最大化"每 FLOP 的智能产出"。每一代的核心改进都围绕一个问题：如何让更多参数在更少计算中发挥作用？

3.1 趋势一：从"全量计算"到"选择性计算"

架构趋势
代际FFN 层注意力层思路
V3MoE（256 选 8）全序列 MLAFFN 已稀疏，注意力仍全量
V3.2MoE（256 选 8）Indexer top-2048注意力也开始稀疏化
V4MoE（更大规模）CSA top-k + HCA 压缩注意力全面稀疏 + 多粒度

趋势总结：DeepSeek 将 MoE 的"选择性激活"思想从 FFN 层推广到了注意力层。V3 时代只有 FFN 是稀疏的（256 个专家选 8 个），注意力仍然是 O(n²) 全量计算。到 V4，注意力也变成了"选择性计算"——只关注最相关的 token 子集。

3.2 趋势二：从"精度统一"到"精度分层"

性能趋势 
V3:   存储 bf16/fp8 可选  |  计算 bf16/fp8 可选  |  KV cache bf16
V3.2: 存储 fp8 only      |  计算 fp8 + 关键路径 fp32  |  KV cache fp8
V4:   存储 fp8 only      |  计算 fp8 + 关键路径 fp32  |  KV cache fp8 分层压缩

设计哲学："低精度存储，高精度计算"。具体来说：

  • 权重存储：FP8（省显存 50%）
  • KV Cache：FP8（省显存 50%，V4 进一步压缩到 10%）
  • 关键计算路径：提升到 FP32（MLP 激活、路由分数、logits、残差加法）

这不是矛盾，而是精细的工程权衡：存储是带宽瓶颈，用低精度省带宽；计算是精度瓶颈，用高精度保质量。

3.3 趋势三：从"工程 trick"到"原生可训练"

方法论趋势
技术V3.2 的做法V4 的做法
稀疏注意力推理时才稀疏（训练仍全量）原生可训练（NSA），训练时就稀疏
残差优化工程上融入 Norm 层理论上施加流形约束
FP8推理量化全流程原生 FP8 训练

趋势总结：V3.2 的优化更像"推理阶段的工程 hack"，V4 则将这些优化"前移"到训练阶段，让模型从一开始就学会在稀疏/低精度条件下工作。这是从"训练后优化"到"训练时优化"的范式转变。

3.4 趋势四：双模型策略（Pro + Flash）

商业趋势

V4 首次推出双版本：

  • Pro（1.6T/49B）：追求极致质量，对标 Claude Opus / GPT-5.5
  • Flash（284B/13B）：追求极致效率，83.6 tokens/s，适合高吞吐场景

这反映了 DeepSeek 从"单一模型打天下"转向"质量-效率帕累托前沿"的产品策略。

四、推理性能优化归因分析

核心问题：V4 Pro 在 1M token 上下文下，推理 FLOPs 仅为 V3 的 27%，KV cache 仅为 10%。这些收益从哪来？

4.3 各代贡献拆解

V3 奠定的基础（仍在 V4 中发挥作用）

  • MLA（Multi-head Latent Attention）：KV 低秩压缩，将每层 KV 维度从 n_heads × head_dim 压缩到 kv_lora_rank=512
  • DeepSeekMoE：256 专家选 8，FFN 层计算量降为 ~3%
  • Auxiliary-loss-free 负载均衡：不引入辅助损失就能保证专家利用率

V3.2 的关键突破（V4 的技术原型）

  • Indexer：首次证明"注意力也可以稀疏化"，top-2048 选择机制
  • FP8 KV Cache：首次将量化从权重扩展到 KV cache
  • 残差融合 Norm：解决 FP8 下残差累积误差问题
  • 移除 bf16 回退：确认 FP8 作为唯一推理精度的可行性

V4 的系统化创新

  • CSA + HCA 混合：将 V3.2 的单一 Indexer 升级为多粒度稀疏注意力
  • 原生可训练稀疏：训练时就使用稀疏注意力，而非推理时才裁剪
  • 流形约束残差：将 V3.2 的工程方案理论化
  • 规模扩展到 1.6T：在效率保证下大幅扩参数

五、DeepSeek 优化方法论总结

方法论框架：三层优化金字塔

        ┌─────────────────────┐
        │   训练方法创新       │  ← V4: 原生稀疏训练、On-policy Distillation
        │  （改变模型学到什么） │
        ├─────────────────────┤
        │   架构创新           │  ← V3→V4: MoE扩展、混合稀疏注意力、流形残差
        │  （改变计算怎么流动） │
        ├─────────────────────┤
        │   工程优化           │  ← V3.2: FP8量化、kernel优化、缓存策略
        │  （改变硬件怎么执行） │
        └─────────────────────┘

DeepSeek 的独特之处：三层同时推进，且下层验证上层。V3.2 先在工程层验证 Indexer 可行，V4 再在架构层系统化为 CSA/HCA，最后在训练层实现原生可训练。这种"自底向上验证、自顶向下设计"的方法论值得学习。

六、未来趋势预判

基于 V3→V4 的演进轨迹，可以预判 DeepSeek 下一步可能的方向：

  1. 注意力进一步稀疏化——当前 top-k=2048 可能继续降低，或引入动态 k（简单 query 用更少 token，复杂 query 用更多）
  2. MoE 专家数继续扩大——从 256 到 512+，总参数可能突破 10T，但激活参数保持在 50-100B
  3. 多模态融合——V4 的 1M 上下文 + 稀疏注意力天然适合处理图像/视频等长序列模态
  4. 推理时计算自适应——根据 query 难度动态调整激活专家数和注意力稀疏度
  5. 端侧部署——Flash 版本（284B/13B）已经接近端侧可部署的规模，下一步可能推出更小的 Nano 版本
