二、架构变化详解

2.1 注意力机制：从 MLA 到混合稀疏注意力

核心创新

V3: Multi-head Latent Attention (MLA)

  • 所有层使用相同的 MLA 机制
  • KV 通过低秩压缩（kv_lora_rank=512）
  • 全序列注意力，复杂度 O(n²)
  • 128K 上下文是实际上限

V4: Hybrid Sparse Attention

  • CSA（Compressed Sparse Attention）：选择 top-k 压缩 token + 滑动窗口
  • HCA（Heavily Compressed Attention）：极度压缩的全局注意力
  • 两种机制交替使用于不同层
  • 1M token 上下文，复杂度接近 O(n)

V4 的混合注意力设计哲学：

  1. CSA 层：通过 Indexer 机制（类似 V3.2 中的原型）选出最相关的 token 子集做精细注意力，同时保留局部滑动窗口确保近距离信息不丢失
  2. HCA 层：将整个序列压缩为极少量的"摘要 token"，提供全局上下文感知，计算量极低
  3. 交替堆叠：CSA 和 HCA 层交替排列，兼顾局部精度和全局视野

2.2 流形约束残差连接（Manifold-Constrained Residual）

训练创新

V3 的问题：标准 Pre-Norm 残差连接在超深网络（61 层）+ FP8 训练中，残差流的数值范围会逐层膨胀，导致训练不稳定。

V4 的解决方案：对残差连接施加流形约束（manifold constraint），确保残差流始终在一个受控的低维流形上演化。具体实现上，将残差加法融入归一化层内部，在 float32 精度下完成，避免 FP8 累积误差。

效果：支撑了 1.6T 参数规模的稳定训练，同时允许更激进的量化策略。

这一设计在 V3.2-Exp 的代码中已有雏形（RMSNorm 融合残差加法），V4 将其系统化并加入了理论约束。

三、关键实现差异

3.1 注意力实现对比

# V3: MLA 全序列注意力（简化伪代码）
def attention_v3(q, k, v, mask):
    # 所有 token 参与注意力计算
    scores = einsum("bshd,bthd->bsht", q, k) * scale
    scores = scores + mask  # causal mask
    attn = softmax(scores)
    return einsum("bsht,bthd->bshd", attn, v)

# V4: 混合稀疏注意力（简化伪代码）
def attention_v4_csa(q, k, v, indexer, window_size):
    # Step 1: Indexer 选出 top-k 相关 token
    topk_indices = indexer(q, k)  # 选出 ~2048 个最相关位置

    # Step 2: 滑动窗口内的 token 也保留
    window_indices = get_local_window(window_size)

    # Step 3: 只在选中的 token 上做注意力
    selected = union(topk_indices, window_indices)
    scores = sparse_attention(q, k[selected], v[selected])
    return scores

def attention_v4_hca(q, k, v, compression_ratio):
    # 将整个序列压缩为少量摘要 token
    compressed_kv = compress(k, v, ratio=compression_ratio)
    scores = full_attention(q, compressed_kv)
    return scores

3.2 KV Cache 策略对比

V3: 全量 KV Cache

  • 存储所有历史 token 的 KV
  • 通过 MLA 低秩压缩减少维度（kv_lora_rank=512）
  • 128K 上下文 ≈ 每层 128K × 512 × 2bytes
  • 61 层总计 ≈ 8GB/batch

V4: 分层压缩 KV Cache

  • CSA 层：只缓存 top-k 选中的 token（FP8）
  • HCA 层：只缓存压缩后的摘要 token
  • 1M 上下文仅需 V3 的 10% 显存
  • 支持 1M token 在单节点部署

3.3 DeepSeek Sparse Attention (DSA) 核心机制

V4 的稀疏注意力（DSA）是从 V3.2-Exp 的 Indexer 演化而来，但更加系统化：

组件V3.2-Exp (Indexer)V4 (DSA)
选择机制单一 top-k（2048）top-k + 滑动窗口 + 压缩全局
应用范围所有层统一CSA/HCA 层交替
训练方式推理时才稀疏原生可训练（Natively Trainable）
硬件对齐通用实现硬件对齐的 kernel 设计
理论基础工程启发式NSA 论文（arxiv:2502.11089）

七、关键洞察

  1. V3.2-Exp 是 V4 的技术预演——Indexer、FP8 KV cache、残差融合 Norm 这三个 V3.2 的核心变化，在 V4 中被系统化为 CSA/HCA 混合注意力 + 流形约束残差。理解 V3.2 的代码就是理解 V4 架构的最佳入口。
  2. "稀疏注意力"是 V4 的核心竞争力——不是简单的 top-k 裁剪，而是原生可训练的、硬件对齐的稀疏机制。这让 1M 上下文从"理论可行"变为"实际可用"。
  3. 规模扩大但效率更高——V4 参数量是 V3 的 2.4 倍，但长上下文推理成本反而更低。这打破了"更大模型 = 更贵推理"的线性关系。
  4. 双模型策略（Pro + Flash）——V4 首次推出轻量版 Flash（284B/13B），覆盖高吞吐低成本场景。V3 时代只有单一模型。
  5. 开源策略更激进——MIT 许可证（V3 的模型协议更严格），权重完全开放，推动生态建设。
