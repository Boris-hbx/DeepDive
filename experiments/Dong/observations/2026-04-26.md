## 最关注的事

### 1. Introducing GPT-5.5

**内容简介**
OpenAI 发布 GPT-5.5，定位为其当前最强模型。官方着重强调 coding、研究、跨工具数据分析等复杂任务的能力提升，速度也有改进。此前独立的代码模型（Codex）线整体合并进主线。

**关键要点**
- 不是渐进升级，是底座代际换代
- 模型层显式把"在电脑上替人干活"（agentic coding + computer use）作为核心能力来打磨
- 多模型线（base / code / vision）走向统一

**对我们的启示**
- 应用层做的 prompt 链 / 工具链拼装的部分能力可能被模型层吸收，**3-6 个月内要重新评估自己的 agent 框架定位**
- DeepDive 这周做的 LLM 抽象层得到验证——"能换 backend 的设计"是必要而非过度
- spec 默认 Claude Sonnet 4.6 之外，可把 GPT-5.5 加入候选 backend 做 A/B（API 公开后）

### 2. GPT-5.5 System Card

**内容简介**
OpenAI 随 GPT-5.5 发布的官方技术补充文档。按惯例包含模型能力 benchmark（含 agentic 任务）、安全评估、红队测试结果，是了解新模型能力边界与已知风险的一手材料。

**关键要点**
- 是模型 agentic 能力的**一手数据源**，不是 PR 通稿
- 大概率含 SWE-Bench、tool use eval、computer use eval 等关键基准
- 信息密度通常远高于公告本身

**对我们的启示**
- 团队内部应建立"读 System Card 不读公告"的习惯，同样适用于 Anthropic / DeepMind 等家
- DeepDive 的 ranker prompt 可加一条"系列发布中识别 System Card 类技术文档优先"的规则
- "公告 + System Card + how-to" 这种系列发布会霸占 top 3，未来可考虑把"系列发布合并为 1 条事件"作为去重优化

### 3. A pelican for GPT-5.5 via the semi-official Codex backdoor API (Simon Willison)

**内容简介**
Simon Willison 通过 OpenAI Codex 客户端的半官方接口绕道试用 GPT-5.5（官方 API 尚未公开），评价模型反应快速高效。文章同时指出 OpenClaw 等第三方 agent 工具借 Codex 订阅接口调用模型，正与 OpenAI 平台方产生摩擦。

**关键要点**
- 顶级模型发布 ≠ 立即可用：API 与产品发布有时间差
- 第三方 agent 工具的**接入策略**成为新博弈点
- "订阅接口被 agent 工具反向调用"这种灰色地带正在被关注

**对我们的启示**
- 选模型不能只看质量，**API 通道的稳定性 + 合规性同等重要**——这正是我们这周 yibuapi 踩坑的更宏观版
- DeepDive spec 关于"信息源选择"的判断应延伸到"API 选择"：每个备选 backend 评估"接入合规性 + 持续性"
- 短期内**不要押注 GPT-5.5 作为 DeepDive 主 backend**（API 未开放）；等 GA 再说

### 5. DeepSeek V4 — almost on the frontier, a fraction of the price (Simon Willison)

**内容简介**
DeepSeek 发布 V4 系列，1.6T 参数开源 MoE 架构（激活仅 49B），性能逼近前沿模型但价格远低。Simon 评测重点是性价比。

**关键要点**
- 国产开源模型在"前沿能力 vs 价格"维度持续紧逼
- MoE 让本地部署 + 低成本推理可行
- 通用迭代，agentic 能力提升不显眼

**对我们的启示**
- **下周可立即做的实验**：跑一次"DeepSeek V4 替代 Claude" 的 DeepDive 同源对比 brief（抽象层这周刚做好正好用）
- 成本对比目标：DeepSeek V4 vs Claude Opus 4.6 的 token 总成本差应在 1/10 量级
- 决策门槛：如果质量 ≥ 80% 且成本 ≤ 20%，**spec 默认 backend 应重新选**

### 8. Codex 合入 GPT-5.5 (Romain Huet, OpenAI)

**内容简介**
OpenAI 高管 Romain Huet 确认 Codex 不再是独立模型线，所有代码相关能力合并进 GPT-5.5。

**关键要点**
- 模型版本管理对用户简化
- 但模型选型从此 = 一体化战略选型，不再能"只选代码模型"

**对我们的启示**
- **抽象层设计前瞻**：未来 model id 趋少（不再分 -code / -vision / -reasoning 子型），抽象层**不要绑死"按用途选模型"逻辑**
- 当前 `LLMClient.complete()` 接口干净，没有 specialized 路径——**没踩这个坑**，回头看是对的
- 后续观察指标：Anthropic / Google 是否跟进"一体化"
