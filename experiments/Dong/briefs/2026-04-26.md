## 最关注的事

### 1. OpenAI 发布 GPT-5.5，统一 agentic coding 与 computer use

OpenAI 把此前分离的代码模型合并进 GPT-5.5，官方给出的关键词是 "strong gains in agentic coding, computer use, and any task on a computer"——意味着模型层把"在电脑上替人干活"作为一类核心能力来打磨。对 agent 工程化的影响是：底座模型的 agentic 能力在加速合并，原先靠应用层拼装的 agent 流水线，可能要重新评估哪些步骤能下沉到模型本身。

来源：[Quoting Romain Huet](https://simonwillison.net/2026/Apr/25/romain-huet/)

### 2. OpenAI 发布 GPT-5.5 prompting guide：建议从零重写 prompt，而非迁移

OpenAI 官方明确建议把 GPT-5.5 当作"独立模型"而非旧版 drop-in 替换，针对老版本积累的 prompt 优化经验**可能失效**，推荐从头构建 prompt。这条对所有在生产里用 OpenAI 模型的团队都是直接成本——如果迁移，prompt 库要重审；如果不迁移，要评估能力差距是否可接受。

来源：[GPT-5.5 prompting guide](https://simonwillison.net/2026/Apr/25/gpt-5-5-prompting-guide/)
