### 1. Behind the Scenes Hardening Firefox with Claude Mythos Preview

Mozilla 利用 Claude Mythos preview 对 Firefox 代码库进行大规模自动化安全审计，发现并修复了数百个漏洞，包括存在 15-20 年的历史缺陷。关键突破不仅来自模型能力提升，更在于工程侧的「驾驭」技术——多模型堆叠、规模化扫描、噪声过滤流水线。Mozilla 指出数月前 AI 安全报告还以低质量 slop 为主，如今信噪比已发生质变。Firefox 现有纵深防御机制拦截了大量利用尝试，说明 agent 审计与传统安全架构形成互补。

来源：[Behind the Scenes Hardening Firefox with Claude Mythos Preview](https://simonwillison.net/2026/May/7/firefox-claude-mythos/#atom-everything)

<summary>延伸思考</summary>

**关键要点**

- 模型能力提升 + 工程化编排（steering / scaling / stacking）双重驱动，信噪比从「slop」跃升至可直接修复
- 发现含 15-20 年历史的深层漏洞，说明 agent 审计覆盖面超越人工常规 review
- Firefox 纵深防御拦截了大量 exploit 路径，agent 审计与防御架构互补而非替代
- 从每季度 20-30 个安全 bug 的基线看，agent 带来的增量修复规模显著

**对我们的启示**

- 安全审计类 agent 已从概念验证进入生产级应用，团队可评估对自有代码库做类似规模化扫描的 ROI
- Mozilla 的多模型堆叠 + 噪声过滤流水线是可复用的 agent 编排模式，值得拆解其 harness 架构
- AI 安全报告质量拐点已到，开源项目维护者需更新对 AI 报告的分类策略——不能再一律视为 slop
- 对于有大量历史代码的项目，agent 审计的优先级应高于仅对新代码做 review，历史债务是高价值目标

</details>

### 2. Agent pull requests are everywhere. Here’s how to review them.

GitHub 官方博客发布针对 agent 生成 PR 的 code review 指南，聚焦 agent PR 常见问题藏匿点、技术债识别方法及审查策略。随着 Copilot agent 和第三方 coding agent 大规模产出 PR，人类 reviewer 的角色从「逐行写码」转向「质量守门」，review 方法论需要相应升级。该指南代表平台方对人机协作流程的正式规范化尝试。

来源：[Agent pull requests are everywhere. Here’s how to review them.](https://github.blog/ai-and-ml/generative-ai/agent-pull-requests-are-everywhere-heres-how-to-review-them/)

<details>
<summary>延伸思考</summary>

**关键要点**

- agent PR 的问题模式与人类 PR 不同：倾向于隐藏技术债而非语法错误，需要针对性审查策略
- GitHub 作为平台方出指南，意味着 agent PR 已从早期尝鲜进入主流工程实践阶段
- review 重心从代码正确性扩展到架构合理性、依赖引入、过度抽象等 agent 常见倾向

**对我们的启示**

- 团队应建立 agent PR 专属 review checklist，重点关注不必要的依赖引入、过度工程化、上下文理解偏差等 agent 特有模式
- CI/CD 流水线中可增加针对 agent PR 的自动化检查（如依赖变更审计、复杂度指标门禁），减轻人工 review 负担
- reviewer 技能需从「能写好代码」向「能判断 agent 产出质量」迁移，这对团队培训和 onboarding 有直接影响

</details>

### 3. Improving token efficiency in GitHub Agentic Workflows

GitHub 官方博客分享其在每个 PR 上运行的 agentic workflow 如何悄然累积高额 API 账单的经验。团队通过对生产环境 workflow 做 instrumentation、定位低效环节，并构建专门的 agent 来修复 token 浪费问题。文章带有实际生产数据和具体优化手段，是目前少见的一线大规模 agentic 系统成本治理实践公开。

来源：[Improving token efficiency in GitHub Agentic Workflows](https://github.blog/ai-and-ml/github-copilot/improving-token-efficiency-in-github-agentic-workflows/)

<details>
<summary>延伸思考</summary>

**关键要点**

- 每个 PR 触发的 agentic workflow 在规模化后 token 成本会隐性膨胀，需主动 instrumentation 才能发现
- 优化路径不只是压缩 prompt，而是用 agent 自身来诊断和修复低效环节
- GitHub 以自身生产环境为案例，提供了可量化的成本治理参考基线

**对我们的启示**

- 团队若在 CI/CD 中嵌入 agentic 流程，应尽早建立 token 用量的可观测性体系，避免账单失控后才被动响应
- 用 agent 优化 agent 的思路值得借鉴——将 token 效率治理本身也做成自动化闭环，而非人工逐条审 prompt
- 评估自有 agentic workflow 时可参照 GitHub 公开的数据做横向对比，判断当前效率水位是否合理

</details>
