# 软件工程每日 Brief — 2026-04-28

## 最关注的事

### 1. GitHub Copilot 开始转向按使用量计费
GitHub 宣布自 6 月 1 日起，Copilot 使用将消耗 GitHub AI 信用点数，正式从订阅制转向按用量付费。这一计费模式调整将直接影响大量开发者和企业的成本结构，也为 AI 编码辅助工具的定价演进提供了重要参考。 [原文](https://github.blog/news-insights/company-news/github-copilot-is-moving-to-usage-based-billing/)

### 2. GitHub 紧急修复 git push 管道中的关键远程代码执行漏洞
GitHub 在一例严重漏洞被发现后，两小时内完成验证、修复与排查，并确认无野外利用。该漏洞影响 git push 管道，对代码托管基础设施的安全构成直接威胁，突显了软件供应链持续面临的攻击风险。 [原文](https://github.blog/security/securing-the-git-push-pipeline-responding-to-a-critical-remote-code-execution-vulnerability/)

### 3. OpenAI 开源 Codex 编排规范 Symphony
OpenAI 发布开源规范 Symphony，可将 Issue 跟踪器转变为常驻 AI 代理系统，旨在减少上下文切换、提升工程产出。若被主流平台广泛采纳，有望重塑开发者与项目管理工具之间的协作方式。 [原文](https://openai.com/index/open-source-codex-orchestration-symphony)
