# Agentic SE 深度卡片：6 个早期重点候选

- 调研日期：2026-04-26
- 数据快照时间：2026-04-26
- 候选来源：`2026-04-26-radar.md` 中调研 agent 推荐的 Top 6 早期项目
- 调研方法：gh CLI 拉 commit/issue/contributor 元数据 + WebFetch 看 README/官网，约 30 次工具调用
- 关键结论：6 个里只有 **Sem/Weave** 和 **Emdash** 同时满足"高速迭代 + 健康社区 + 清晰差异化"，建议重点跟踪

---

## 1. Live-SWE-agent

**项目本质**：一篇学术论文（arXiv 2511.13646）的代码 release，以"agent 在 runtime 自我修改"为核心 selling point，跑 SWE-bench Verified 拿 79.2%（Claude Opus 4.5）。基于 mini-swe-agent。

**作者/团队背景**：OpenAutoCoder 组织，与 AutoCodeRover 同源（伊利诺伊大学 / 新加坡国立 SE 研究圈）。学术背景，非创业团队。

**活跃度**：
- 376 stars / 38 forks / 2 contributors
- 30 天 0 commits，最后 commit 2026-01-19（**停滞 ~3 个月**）
- 唯一 release v1.0.0 在 2025-11
- 9 个 open issues 几乎无回复（"GPT-5 不会创建 tools"挂了 4 个月）

**技术差异点**：相比 SWE-agent 主打 "self-evolving"，但 README 没讲清自我演化的具体机制。本质是 mini-swe-agent + 一些 runtime hook。

**路线图**：无。论文已发，repo 进入维护期。

**风险信号**：典型"论文配套代码 dump"模式 —— 发完论文就停。无 release 节奏、issue 无人响应、commit 全是文档微调。

**判断**：**低**。学术 PoC 已交付，没有产品化意图。SWE-bench 榜单出现新一名再回头看一次即可。

## 2. Sem + Weave（Ataraxy Labs）

**项目本质**：把 git 的 line-based 操作升级成 entity-based（基于 tree-sitter 的函数/类/方法粒度）。Sem 做 diff/blame/impact，Weave 做 merge driver。两者共享 sem-core 解析层。

**作者/团队背景**：Ataraxy Labs，2025 年成立，创始人含 Rohan。自我定位 "agent-native software development substrate"。已有 4 个产品在出货（sem / weave / inspect / opensessions）—— 产品节奏极强，疑似有 backing 但无公开融资。

**活跃度**：
- **Sem**：1897 stars / 100+ commits/30 天 / 17 contributors / 每周发版（v0.3.20 → v0.3.24，4 月一个月发 5 个 minor）
- **Weave**：962 stars / 73 commits/30 天 / 10 contributors / 半月内 v0.2.7 → v0.3.2
- Issue 互动质量高：rs545837（核心维护者）当天回复 breaking-change 投诉并承诺加 cargo-semver-checks 到 CI

**技术差异点**：与 SWE-agent / OpenHands 这类 agent runtime **完全不同维度** —— 是给 agent 用的**底层基础设施**。Mergiraf 是最直接的同类，benchmark 显示 31/31 vs 26/31 全胜。Rust 实现，支持 26 种语言。

**路线图**：WASM 支持（issue #34 高讨论）、与 diff-so-fancy 集成（#78）、MCP server 已就位。

**风险信号**：
- v0.3.24 当成 minor 发其实是 breaking —— semver 纪律不严
- 高速迭代 + 极小核心团队（实质 1 个核心 + 长尾 17 contributors），有 bus factor 风险

**判断**：**高**。这是少数把 "agent-native infra" 真正做成产品的团队，不卷 agent 编排，做被所有 agent 依赖的底层。

**值得挖的 angle**：
1. **Sem/Weave 是否会被 Cursor / Claude Code / Codex 集成进 diff/merge 流程** —— 一旦有大厂签约，估值跳变
2. **Inspect（code review）和 opensessions（tmux）是 Ataraxy 产品矩阵其余两块**，连起来看就是"agent 全栈 SE 基础设施"，是否构成新的标准栈

## 3. OpenHarness（HKUDS）

**项目本质**：港大数据科学实验室（HKUDS，做 LightRAG / GraphRAG 那个组）的"轻量 agent harness"，自带个人 agent ohmo（接 Feishu/Slack/Telegram，复用你已有的 Claude Code/Codex 订阅跑活）。MIT，**2026-04-01 创建**。

**作者/团队背景**：HKUDS = 港大 Data Intelligence Lab，已有 LightRAG（19k+ stars）、MiniRAG 等成功项目，**学术 + 网红流量打法成熟**。主贡献者 tjb-tech 一人 196 commits（绝对核心）。

**活跃度**：
- 11.3k stars / 1.9k forks（**25 天涨到这个量** —— 典型 HKUDS 上线即热，部分靠组里影响力一波带）
- 38 contributors（含 Hinotoi-agent / Copilot 等 bot 账号，真实人类核心 ~10）
- 100+ commits/30 天，已发 v0.1.6、v0.1.7

**技术差异点**：与 OpenHands 相比更轻、与 Claude Code 相比加了多渠道（Feishu/Slack）和"不耗自己 API key、复用订阅"卖点。本质是"agent 编排框架 + 个人助理参考实现"双层。

**路线图**：dry-run 安全预览、ClawTeam 集成、扩展 provider。

**风险信号**：
- 真实痛点 issue：macOS/Windows 终端 backspace、`/compact` 留 orphan tool_use 块导致 API 400、`--task-worker` 多行 stdin 丢失 —— **典型早期 alpha 体验**
- star/issue 比例严重失衡（11k stars / 8 open issues）—— 大量观望用户 ≠ 真实使用者
- HKUDS 有"开源 6 个月达到峰值就转下个项目"的历史模式（看 LightRAG 现在的维护节奏）

**判断**：**中**。短期高流量但需要观察 90 天看是否进入 LightRAG 那种"高 star + 中等维护"稳态，还是变成弃坑。

**值得挖的 angle**：ohmo 的"复用订阅、不烧 API key"是个**反 OpenAI/Anthropic 商业模型**的角度，如果用户增长真起来会被注意到 —— 跟踪 ohmo 用户使用反馈而非 OpenHarness 框架本身。

## 4. SE-Agent（JARVIS-Xs / QuantaAlpha）

**项目本质**：自称 SWE-bench Verified 80% Top1、NeurIPS 2025 论文配套代码。核心思想是 trajectory-level 演化（Revision/Recombination/Refinement）。

**作者/团队背景**：QuantaAlpha 自称"elite research team"，主页 quantaalpha.github.io。无知名学术机构挂靠，无清晰商业实体。

**活跃度**：
- 268 stars / 5 contributors
- **30 天 0 commits**，最后 commit 2025-09-22（**停滞 7 个月**）
- 0 release
- 唯一一个有意义的 issue 是"无法复现 80% 性能"挂 5 个月无人理

**技术差异点**：路径级演化思路在论文层面有意思（trajectory-level vs agent-level），但实现细节不开源完整。

**风险信号**：**几乎确定已弃坑**。issue #15 "Unable to Reproduce 80%" 无回复 = 论文复现性存疑 + 团队不在了。

**判断**：**低**。除非他们出 v2 论文，否则不值得跟。

## 5. Emdash（YC W26）

**项目本质**：桌面端 ADE（Agentic Development Environment），核心机制是给每个 agent 一个 git worktree 并行跑，支持 23 个 CLI agent（Claude Code / Cursor / Copilot / Cline 等）。Provider-agnostic，可远程 SSH。

**作者/团队背景**：**YC W26 批次**。德国创始团队 @rabanspiegel + @arnestrickmann + @jschwxrz，三人都是高频 contributors（合计 ~3600 commits）。是这次 6 个里**最像"正经创业公司"的项目**。

**活跃度**：
- 4125 stars / 376 forks / **87 contributors**
- 100+ commits/30 天
- 2 个 release（v0.1.6、v0.1.7 4 月内）
- issues + PR 近一周内还在每天涌入（编号已到 #1774）
- **最健康的社区**

**技术差异点**：与 conductor.build（同样做 parallel agent + worktree 的 YC 项目）正面竞争；与 Cline/Cursor 不同 —— 它是 **agent 编排器/容器**，而不是 agent 本身。Provider-agnostic 是关键卖点。

**路线图**：从 issue 看正在加 Zed Nightly、settings UI、navigation 等 polish —— 产品化阶段。

**风险信号**：
- 商业化迹象明显（YC W26、官网 emdash.sh、社交账号体系完整）—— **未来很可能 freemium 化**
- 所有 issue/PR 由核心团队 review，PR 队列堆积速度快于合并速度
- 与 conductor.build 竞争激烈，谁先拿到 Cursor-tier 用户量谁活下来

**判断**：**高**。它是这 6 个项目里**唯一处于"产品化 + 高速增长 + 健康社区"三重共振**的。

**值得挖的 angle**：
1. **parallel agent 是 agentic SE 的下一个 UX 形态**（继 chat、inline 之后的第三种），Emdash + conductor 的对比本身就是周报常驻话题
2. **YC W26 demo day**（应在 2026-03/04 已发生）后是否有融资公告/估值 —— 查 TechCrunch / YC 官博

## 6. Trae Agent（ByteDance）

**项目本质**：字节 Trae IDE 的 CLI agent 部分开源版，定位"research-friendly architecture for SE tasks"，多 provider 支持。

**作者/团队背景**：字节跳动，背靠 Trae IDE 商业产品（trae.ai）。

**活跃度**：
- 11.4k stars / 1.2k forks / 49 contributors
- **30 天 0 commits**，最后 commit 2026-02-05（**停滞 ~80 天**）
- **0 release**（从未正式 release）
- Open issues/PR 还在涨但无人 merge

**技术差异点**：与 SWE-agent 同代，无明显架构创新；定位"研究友好"实际上意味着没拿来当产品做。

**风险信号**：典型"大厂 OSS 装门面"模式 —— 开源是 PR 行为，不是工程承诺。stalled 80 天 + 0 release + open PR 堆积 = **半 archive 状态**。

**判断**：**低**。要看字节在 agentic SE 的真动作，应该跟 Trae IDE（trae.ai 闭源产品）的更新而不是这个 repo。在 radar 标记"已停滞"并降权。

## 汇总表

| 项目 | 跟踪价值 | 一句理由 | 推荐周扫描频率 |
|------|---------|----------|---------------|
| Live-SWE-agent | 低 | 论文配套代码，已停滞 3 个月 | 季度回看 |
| **Sem + Weave** | **高** | agent-native infra 路线最清晰，每周发版，可能被大厂集成 | **每周**（重点看 release notes 和 issue 里的集成讨论） |
| OpenHarness | 中 | 25 天 11k stars 但 UX 仍 alpha，HKUDS 历史上有半年弃坑模式 | 每 2 周（90 天后重评） |
| SE-Agent | 低 | 7 个月无 commit，复现性投诉无人答 | 不跟 |
| **Emdash** | **高** | YC W26 + 三创始人 + 87 contributors + 健康社区，parallel agent UX 代表作 | **每周**（关注 release + 融资动态 + vs conductor.build） |
| Trae Agent | 低 | 80 天无 commit，字节精力已回闭源 Trae IDE | 季度回看（或转跟 Trae IDE 本体） |

## 关键洞察

1. **6 选 2**：只有 Sem/Weave 和 Emdash 值得放到周跟踪盘里，其他 4 个掉进"学术 dump / 大厂 OSS 装门面 / 已弃坑"三种 anti-pattern
2. **infra vs UX 两条线最值得跟**：Sem/Weave 代表"agent 调用的底层版本控制基础设施"，Emdash 代表"多 agent 并行 UX"。这两条线在 agentic SE 的下一阶段最有 narrative
3. **数据快照需补"年龄"维度**：原 radar 没区分"11k stars 是 1 年攒的还是 25 天暴涨的"。OpenHarness 和 Trae Agent 同样 11k stars，意义完全不同。下次扫描应在 stage 字段加上 age（首次 commit 距今天数）
4. **下次扫描可补的角度**：
   - conductor.build（Emdash 的直接竞品）应纳入下一轮
   - Ataraxy Labs 另两个产品（Inspect / opensessions）应单独成卡
   - HKUDS 组里其他 active 的 agent 项目可一并扫
