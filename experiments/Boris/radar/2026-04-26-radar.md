## 调研方法注释

**实际跑过的检索通道：**

1. GitHub Topic 页 ~20 个：`coding-agent`、`ai-coding-assistant`、`swe-agent`、`code-agent`、`aider`、`autonomous-coding`、`copilot-alternative`、`cursor-alternative`、`agentic-coding`、`devin`、`agentic-ai`、`claude-code`、`codex`、`openhands`、`cline`、`llm-coding`、`llm-agent`、`ai-software-engineer`、`code-review` 等
2. Awesome 列表 8+ 个（见上方"类别 10"）
3. SWE-bench 生态：swebench.com leaderboard、swe-bench/swe-bench GH 组织、Live-SWE-agent leaderboard、Scale SWE-bench Pro leaderboard
4. 关键词搜索：`"open source coding agent"`、`"autonomous coding agent"`、`"AI software engineer"`、`"code agent" framework SDK`、`"agent harness" coding`、`"Show HN" coding agent`、中文 `"代码 agent" 开源 2026`
5. 策展博文：a16z notes on AI apps、Latent Space、Stoneforge open-source agents list、tencentcloud 开源 agent 总览
6. 直接抓 ~50 个 repo 的 owner/stars/lang/last-release 元数据

**已知遗漏角度（建议后续补查）：**

1. **SWE-bench 主榜单的隐藏提交者**：swebench.com leaderboard 表格 JS 渲染，WebFetch 抓不到详细行，需要浏览器/API 拉。建议翻 `swe-bench/experiments` 子目录下所有提交者的 repo 链接
2. **Hugging Face Spaces / model harness**：Devstral、Kimi K2、SERA-32B、CWM 等配套 harness 可能藏在 HF org 而非 GitHub topic
3. **学术 paper repo**：MASAI、Salesforce DEI Agents、SWE-PolyBench (Amazon)、Cline Bench、DPAI Arena (JetBrains)、Context-Bench (Letta)、CORE-Bench、APEX-Agents、MirrorCode 等没逐一抓元数据
4. **国产 OSS 边界模糊**：通义灵码、CodeBuddy、CodeGeeX、智谱、Trae 的"是否完全开源"分歧；本清单只覆盖明确 OSS 的 Trae-Agent、CodeGeeX4、Costrict
