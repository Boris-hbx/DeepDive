### 3. Codex CLI 新增 /goal 自主循环功能

OpenAI 的 Codex CLI 0.128.0 版本新增了 /goal 功能，实现了类似 Ralph loop 的自主循环机制：设定目标后 agent 会持续执行直到自评目标完成或 token 预算耗尽。该功能通过 goals/continuation.md 和 goals/budget_limit.md 两个 prompt 模板实现，在每个 turn 结束时自动注入。这标志着 coding agent 从单轮交互向持续自主执行的重要演进，与 Claude Code 的 /loop 和 Cursor 的 background agent 形成竞争态势。

## 蓝军反驳（Boris 实验扩展，非 spec 契约）

### 反驳 #1（针对 HERMES.md 计费路由 bug）

**原文观点**：「这暴露了 agentic coding 工具在计费透明度方面的隐患——当工具自主生成 commit 时，用户可能在不知情的情况下触发不同的计费逻辑。」

**蓝军视角分析**：从单个 bug 跳跃到「计费透明度隐患」是过度外推。路由异常可能是字符串匹配的工程 bug（如内部标记被误触发），而非有意的计费设计。摘要指出「Anthropic 尚未正式回应」，但将沉默解读为默认有问题是典型的证据缺失谬误——大公司通常需要内部调查后才会公开回应。此外，1237 点赞反映的是社区情绪强度，不等于技术事实的严重程度。

### 反驳 #2（针对 Claude Code 竞品关键词审查）

**原文观点**：「如果编码 agent 会根据代码内容选择性地拒绝服务，开发者如何信任它处理敏感项目？」

**蓝军视角分析**：这个结论建立在单一 Twitter 报告之上，缺乏独立复现。AI 工具的行为报告在社交媒体上极易被放大和误读——用户可能遇到的是 rate limit、内容安全过滤或随机拒绝，而非针对竞品名称的定向审查。摘要将此事与 HERMES.md bug 并列，暗示存在系统性模式，但两个未经证实的独立报告不构成模式。在没有可复现的 reproduction steps 之前，「选择性拒绝服务」仍是未经检验的假设。

### 反驳 #3（针对 Codex CLI /goal 自主循环）

**原文观点**：「这标志着 coding agent 从单轮交互向持续自主执行的重要演进。」

**蓝军视角分析**：自主循环并非新概念——AutoGPT（2023）、Claude Code /loop、Cursor background agent 都已实现类似机制。Codex CLI 的 /goal 是跟进而非开创。更关键的是，摘要提到 agent「自评目标完成」作为终止条件，但自评可靠性是 agentic 系统的已知弱点：agent 判断自己「完成了」不等于目标真正达成。将 token 预算耗尽作为兜底机制，本质上承认了自评不可靠，这恰恰削弱了「重要演进」的叙事。
