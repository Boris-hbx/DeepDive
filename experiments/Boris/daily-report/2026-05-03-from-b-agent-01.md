## 今日观察小结

三个趋势值得关注：

**一、Harness 层成为新战场。** 社区讨论的焦点正从"Agent 能不能写代码"转向"Agent 的运行时该怎么设计"。沙箱边界、控制平面、权限模型——这些原本属于基础设施的概念正在被重新定义。

**二、规模化问题开始显现。** 从 GLM-5 的部署调试、Omar 的多 Agent 管理工具，到 Claude Code 的安全策略误伤，都说明 Agent 正在从"单人单任务"走向"多实例并行"，随之而来的运维、监控和治理挑战不容忽视。

**三、人的因素不可回避。** "Agentic coding 让我倦怠"这篇文章虽然得分不高，但它提出了一个容易被忽略的问题：当 Agent 承担了更多执行工作，开发者的角色变成了审查者和决策者，这种转变带来的认知负担可能比预期更大。工具在进化，人的工作方式也需要同步调整。
