## 你的 Self-Evolve Agent 可以关注的优化方向

在讨论具体 benchmark 之前，先梳理一下 self-evolve agent 通常需要衡量的几个核心维度，再把本次采集到的业界动态对号入座。

### 核心优化目标拆解

| 维度 | 关键问题 | 可量化的方向 |
|------|---------|-------------|
| 输出确定性 | agent 在相同输入下能否产出稳定、结构化的结果？ | 结构化输出一致性、格式合规率 |
| 记忆与上下文管理 | 多轮演化过程中，agent 能否有效存取和利用历史信息？ | 记忆召回准确率、上下文利用效率 |
| 创造性演化 | agent 的自我改进是否只是微调，还是能产生真正有价值的新方案？ | 创意多样性、方案新颖度 |
| 效率与成本 | 每次演化迭代的 token 消耗、延迟、实际收益比 | token 效率、迭代收敛速度 |

## 最关注的事

### 1. 结构化输出 Benchmark：衡量 agent 演化结果的确定性

对 self-evolve agent 来说，每一轮自我改进的输出必须是可解析、可执行的（比如生成新的 prompt、代码片段、配置）。如果输出格式不稳定，下游的自动化流程就会断裂。这个新 benchmark 专门测试 LLM 产出确定性结构化输出的能力，可以直接用来评估你的 agent 在"自我修改"环节的输出可靠性。

建议用法：在 agent 每次生成改进方案后，用该 benchmark 的评估方法检查输出是否严格符合预期 schema，作为演化质量的门控指标。

来源：[Show HN: A new benchmark for testing LLMs for deterministic outputs](https://interfaze.ai/blog/introducing-structured-output-benchmark)

### 2. Xmemory：结构化记忆 vs RAG，agent 的"长期记忆"该怎么选

Self-evolve agent 的核心能力之一是从历史迭代中学习。这篇论文对比了结构化 AI 记忆、纯 RAG 和混合 RAG 三种方案的表现。如果你的 agent 目前用的是简单的上下文拼接或向量检索，这个 benchmark 能帮你量化评估是否需要切换到结构化记忆方案。

建议 metrics：记忆命中率（历史经验被正确召回的比例）、遗忘率（关键信息在多轮后丢失的比例）。

来源：[Xmemory: Benchmarking Structured AI Memory Against RAG and Hybrid RAG](https://arxiv.org/abs/2604.27906)

### 3. 创造力 Benchmark：agent 的"进化"不只是拟合，还要有新意

Self-evolve 的终极目标不是收敛到局部最优，而是能探索出人类未预见的改进路径。这个 Human Creativity Benchmark 提供了一套评估生成式 AI 创造力的框架，可以借鉴其中的评估维度（新颖性、实用性、意外性）来衡量 agent 每轮演化方案的质量。

建议用法：对 agent 产出的改进方案做"创意评分"，避免陷入重复微调的死循环。

来源：[The Human Creativity Benchmark – Evaluating Generative AI in Creative Work](https://contralabs.com/research/human-creativity-benchmark)

## 针对你的 Self-Evolve Agent 的 Metrics 建议汇总

| Metric | 说明 | 对应 Benchmark/方法 |
|--------|------|-------------------|
| 输出结构合规率 | 每轮演化输出是否符合预定义 schema | Structured Output Benchmark |
| 记忆召回精度 | 历史经验在后续迭代中被正确利用的比例 | Xmemory 评估框架 |
| 方案新颖度 | 新方案与历史方案的差异度，避免原地踏步 | Human Creativity Benchmark 维度借鉴 |
| 迭代收敛效率 | 达到目标性能所需的演化轮数 | 自定义，跟踪每轮 score 变化曲线 |
| Token 成本比 | 每单位性能提升消耗的 token 数 | 自定义，参考 caveman 测试的思路 |
| 回归率 | 新一轮演化是否导致已解决问题复现 | 自定义回归测试集 |

## 今日观察小结

今天采集到的信号虽然不多，但恰好覆盖了 self-evolve agent 的三个关键面：输出稳定性、记忆机制、创造力评估。建议优先引入结构化输出 benchmark 作为最基础的质量门控，再逐步把记忆评估和创意评分纳入演化循环。另外，caveman 测试提醒我们一个容易忽略的点：agent 的自我优化不应只做加法，"做减法"本身也是一种有效的演化方向。
