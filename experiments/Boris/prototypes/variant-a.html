REBUTTAL · 01
「现有 agent 架构中的编排层定位需要重新评估」
模型能力增强 ≠ 编排层即将消失。每次 GPT 重大升级（GPT-3 → 4 → 4o → 5）都伴随相同的"编排层会被吃掉"叙事，但实测的反例是：① 跨长期任务的状态管理仍需应用层（model context 不是无限的）；② 工具链可观测性、人审、合规等需求与模型能力正交，不会因 model 升级而消失；③ 真正被吸收的是 prompt orchestration 中较低阶的拼装，而高阶任务（角色分工、长流程）反而更依赖编排。说"重新评估"是对的，但不应隐含"编排层缩水"。
REBUTTAL · 02
「该文档大概率涵盖 agentic 场景下的 benchmark 数据」
System Card 历史上对 agentic eval 覆盖严重不足。OpenAI 历版 System Card 对 SWE-bench / ToolBench / GAIA 等 agent benchmark 的报告深度，远不如 MMLU / MATH 这类传统能力 benchmark；红队测试也偏重 prompt injection 和有害内容，对长程任务的鲁棒性、self-correction 失败模式几乎不涉及。"大概率涵盖"是过度乐观——更可能的情况是有几个总分但缺细分维度。
REBUTTAL · 03
「API 开放节奏和接入策略对 agent 工程化落地有直接影响」
表述太弱、没说清"直接影响"是什么方向。① 对 OpenAI 一方而言，限制 API 反而强化其 ChatGPT/Codex 端到端控制权，这是商业利好不是利空；② 对 agent 应用方而言，多模型抽象（如 LangChain、LiteLLM、本仓 Dong 的 llm/ 后端层）已经把"单一供应商接入策略"的影响降到很低；③ 真正值得关注的不是 API 何时开，而是定价和 rate limit——那才是落地经济性的硬约束。
