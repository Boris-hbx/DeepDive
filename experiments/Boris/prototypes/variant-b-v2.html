## [01]Introducing GPT-5.5

OpenAI 发布 GPT-5.5，官方定位为其最强模型，重点提升 coding、研究和跨工具数据分析等复杂任务能力，同时强调速度改进。对 agent 工程化的直接含义：底座模型原生的跨工具协作能力增强，此前靠应用层 prompt 编排和工具链拼装补齐的部分 gap 可能被模型层吸收，现有 agent 架构中的编排层定位需要重新评估。

原文观点
现有 agent 架构中的编排层定位需要重新评估
蓝军视角分析
模型能力增强 ≠ 编排层即将消失。每次 GPT 重大升级（GPT-3 → 4 → 4o → 5）都伴随相同的"编排层会被吃掉"叙事，但实测的反例是：(1) 跨长期任务的状态管理仍需应用层（model context 不是无限的）；(2) 工具链可观测性、人审、合规等需求与模型能力正交，不会因 model 升级而消失；(3) 真正被吸收的是 prompt orchestration 中较低阶的拼装，而高阶任务（角色分工、长流程）反而更依赖编排。说"重新评估"是对的，但不应隐含"编排层缩水"。

## [02]GPT-5.5 System Card

OpenAI 发布 GPT-5.5 的 System Card，按惯例包含模型能力基准、安全评估及红队测试结果。该文档大概率涵盖 agentic 场景下的 benchmark 数据，是了解 GPT-5.5 在 agent 任务中能力边界与已知风险的一手材料。作为模型发布后的补充文档，技术细节密度通常高于公告本身。

## [03]A pelican for GPT-5.5 via the semi-official Codex backdoor API

OpenAI 发布 GPT-5.5，面向 Codex 和 ChatGPT 付费用户开放，但官方 API 尚未上线，称需额外安全审查。Simon Willison 通过 Codex 提供的半官方接口绕道实测，评价模型快速高效。文章同时指出 OpenClaw 等 agent 工具经订阅接口调用模型的做法正与平台方产生摩擦，API 开放节奏和接入策略对 agent 工程化落地有直接影响。
