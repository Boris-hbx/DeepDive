课题一 / Topic 1

课题名称（中文）：AI 驱动的自适应攻击面管理

Topic Name (English): AI-Driven Adaptive Attack Surface Management

核心问题 / Core Question

如何在云原生、微服务和持续部署环境下，利用 AI 实现攻击面的实时感知、动态收敛和持续验证？

How can AI be leveraged to achieve real-time perception, dynamic convergence, and continuous verification of attack surfaces in cloud-native, microservice, and continuous-deployment environments?

为什么值得做 / Why It Matters

现代基础设施的攻击面不再是静态的——容器编排、Serverless 函数、API 网关和第三方集成使得暴露面持续变化。源报告中涉及的多类漏洞（配置错误、未授权访问、API 安全缺陷）本质上都是攻击面管理失控的表现。传统的周期性扫描和人工资产盘点已无法跟上 DevOps 的节奏，亟需一种 AI 驱动的、能够自适应感知并主动收敛攻击面的新范式。

The attack surface of modern infrastructure is no longer static — container orchestration, serverless functions, API gateways, and third-party integrations cause continuous exposure changes. The various vulnerability classes covered in the source reports (misconfigurations, unauthorized access, API security flaws) are fundamentally symptoms of uncontrolled attack surface sprawl. Traditional periodic scanning and manual asset inventories can no longer keep pace with DevOps velocity, creating an urgent need for an AI-driven paradigm that adaptively perceives and proactively converges the attack surface.

可能的切入点 / Possible Entry Points

1. 动态资产图谱构建 / Dynamic Asset Graph Construction — 基于运行时流量和编排元数据，自动构建实时资产拓扑图谱

2. 暴露面风险量化模型 / Exposure Risk Quantification Model — 结合 CVSS、可达性分析和业务上下文，对每个暴露点进行动态风险评分

3. 自动化攻击面收敛策略 / Automated Attack Surface Convergence Policies — 利用强化学习生成最优的防火墙规则、网络分段和访问控制策略

4. 持续验证闭环 / Continuous Verification Loop — 将 BAS（Breach and Attack Simulation）与攻击面管理平台集成，形成"感知→收敛→验证"闭环

课题二 / Topic 2

课题名称（中文）：软件供应链深度信任链建模与验证

Topic Name (English): Deep Trust Chain Modeling & Verification for Software Supply Chains

核心问题 / Core Question

如何对软件供应链中的传递性依赖关系建立可形式化验证的信任模型，从而系统性地发现和阻断信任链中的薄弱环节？

How can we build formally verifiable trust models for transitive dependency relationships in software supply chains to systematically discover and block weak links in the trust chain?

为什么值得做 / Why It Matters

源报告中多次提及的第三方组件漏洞、依赖库风险和构建流程安全问题，揭示了一个深层矛盾：现代软件高度依赖开源生态，但对依赖链的信任验证极其薄弱。一个底层库的妥协可以沿依赖图谱向上传播，影响数千个下游项目。当前的 SBOM（软件物料清单）方案仅解决了"可见性"问题，但尚未解决"可信性"问题——我们知道用了什么，但不知道能否信任它。这一课题旨在从可见性迈向可验证的信任。

The third-party component vulnerabilities, dependency risks, and build-process security issues repeatedly mentioned in the source reports reveal a deep contradiction: modern software heavily relies on the open-source ecosystem, yet trust verification of the dependency chain is extremely weak. A compromise in a low-level library can propagate up the dependency graph, affecting thousands of downstream projects. Current SBOM approaches only solve the "visibility" problem but not the "trustworthiness" problem — we know what we use, but not whether we can trust it. This topic aims to move from visibility to verifiable trust.

可能的切入点 / Possible Entry Points

1. 传递性信任衰减模型 / Transitive Trust Decay Model — 研究信任如何沿依赖链衰减，建立数学模型量化 N 层依赖后的残余信任度

2. 构建流程完整性证明 / Build Process Integrity Proofs — 扩展 SLSA 框架，引入零知识证明技术验证构建过程的完整性而不暴露源码

3. 依赖行为异常检测 / Dependency Behavioral Anomaly Detection — 通过沙箱执行和系统调用分析，检测依赖包在安装和运行时的异常行为（如意外网络请求、文件系统越权访问）

4. 信任链可视化与决策支持 / Trust Chain Visualization & Decision Support — 构建交互式依赖信任图谱，辅助安全团队快速定位信任链中的最薄弱节点

课题三 / Topic 3

课题名称（中文）：大模型安全对齐的对抗鲁棒性研究

Topic Name (English): Adversarial Robustness of LLM Safety Alignment

核心问题 / Core Question

如何构建在对抗性输入下仍能保持安全对齐特性的大语言模型防护机制，使其在实际部署中抵御提示注入、越狱攻击和对齐逃逸？

How can we build defense mechanisms for large language models that maintain safety alignment properties under adversarial inputs, resisting prompt injection, jailbreak attacks, and alignment evasion in real-world deployments?

为什么值得做 / Why It Matters

源报告中关于 AI 安全风险的讨论表明，大模型正在被快速集成到各类业务系统中，但其安全对齐机制在对抗性条件下极为脆弱。提示注入攻击可以绕过安全护栏，使模型输出有害内容或执行未授权操作。更深层的问题在于，当前的对齐方法（RLHF、Constitutional AI 等）本质上是统计性的，缺乏形式化的安全保证。随着 LLM 成为关键基础设施的一部分，对齐的对抗鲁棒性将成为一个核心安全课题。

The AI security risk discussions in the source reports indicate that large models are being rapidly integrated into various business systems, yet their safety alignment mechanisms are extremely fragile under adversarial conditions. Prompt injection attacks can bypass safety guardrails, causing models to output harmful content or perform unauthorized operations. The deeper issue is that current alignment methods (RLHF, Constitutional AI, etc.) are fundamentally statistical and lack formal safety guarantees. As LLMs become part of critical infrastructure, adversarial robustness of alignment will become a core security research topic.

可能的切入点 / Possible Entry Points

1. 对齐逃逸的系统化分类学 / Systematic Taxonomy of Alignment Evasion — 建立覆盖提示注入、多轮越狱、编码绕过等攻击向量的完整分类框架，为防御研究提供基准

2. 运行时安全护栏架构 / Runtime Safety Guardrail Architecture — 设计独立于模型本身的外部安全层，通过输入过滤、输出检测和语义一致性校验实现多层防御

3. 形式化对齐验证 / Formal Alignment Verification — 探索将形式化方法（如抽象解释、模型检验）应用于 LLM 安全属性验证的可行路径

4. 红队自动化与对齐压力测试 / Automated Red Teaming & Alignment Stress Testing — 利用对抗性 AI 自动生成越狱攻击样本，构建持续性的对齐鲁棒性评估流水线

📝 总结

以上三个课题从源报告中的具体安全事件和技术方案向上抽象，分别对应三个宏观趋势：

  • •课题一回应的是基础设施复杂性爆炸带来的攻击面失控问题
  • •课题二回应的是软件生态系统化带来的信任传递性风险
  • •课题三回应的是 AI 深度嵌入安全关键系统后带来的对齐脆弱性

三者共同指向一个核心判断：安全研究正在从"修补已知漏洞"转向"构建可验证的信任与鲁棒性"。
