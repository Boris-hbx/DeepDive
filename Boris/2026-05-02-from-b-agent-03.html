## [01]结构化输出 Benchmark：衡量 agent 演化结果的确定性

对 self-evolve agent 来说，每一轮自我改进的输出必须是可解析、可执行的（比如生成新的 prompt、代码片段、配置）。如果输出格式不稳定，下游的自动化流程就会断裂。这个新 benchmark 专门测试 LLM 产出确定性结构化输出的能力，可以直接用来评估你的 agent 在"自我修改"环节的输出可靠性。 建议用法：在 agent 每次生成改进方案后，用该 benchmark 的评估方法检查输出是否严格符合预期 schema，作为演化质量的门控指标。

## [02]Xmemory：结构化记忆 vs RAG，agent 的"长期记忆"该怎么选

Self-evolve agent 的核心能力之一是从历史迭代中学习。这篇论文对比了结构化 AI 记忆、纯 RAG 和混合 RAG 三种方案的表现。如果你的 agent 目前用的是简单的上下文拼接或向量检索，这个 benchmark 能帮你量化评估是否需要切换到结构化记忆方案。 建议 metrics：记忆命中率（历史经验被正确召回的比例）、遗忘率（关键信息在多轮后丢失的比例）。

## [03]创造力 Benchmark：agent 的"进化"不只是拟合，还要有新意

Self-evolve 的终极目标不是收敛到局部最优，而是能探索出人类未预见的改进路径。这个 Human Creativity Benchmark 提供了一套评估生成式 AI 创造力的框架，可以借鉴其中的评估维度（新颖性、实用性、意外性）来衡量 agent 每轮演化方案的质量。 建议用法：对 agent 产出的改进方案做"创意评分"，避免陷入重复微调的死循环。 ---
