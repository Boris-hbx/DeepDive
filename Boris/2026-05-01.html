## [01]Claude Code commit message 触发额外计费路由

有用户发现在 commit message 中包含 HERMES.md 字样时，Claude Code 的请求会被路由到额外用量计费通道。该 issue 在 GitHub 上获得 1237 点赞和 526 条评论，成为近期 Claude Code 最大的信任事件。这暴露了 agentic coding 工具在计费透明度方面的隐患——当工具自主生成 commit 时，用户可能在不知情的情况下触发不同的计费逻辑。Anthropic 尚未正式回应。

## [02]Claude Code 对竞品关键词的异常行为

Theo 在 Twitter 上报告 Claude Code 在 commit message 中检测到 'OpenClaw' 等竞品相关关键词时，会拒绝执行请求或将请求路由到更高计费层级。该帖获得 1267 点赞和 694 条评论，与 HERMES.md 计费 bug 共同构成了本周 Claude Code 最大的争议事件。这对 agentic SE 工具的中立性和可信度提出了严峻质问——如果编码 agent 会根据代码内容选择性地拒绝服务，开发者如何信任它处理敏感项目？

## [03]Codex CLI 新增 /goal 自主循环功能

OpenAI 的 Codex CLI 0.128.0 版本新增了 /goal 功能，实现了类似 Ralph loop 的自主循环机制：设定目标后 agent 会持续执行直到自评目标完成或 token 预算耗尽。该功能通过 goals/continuation.md 和 goals/budget_limit.md 两个 prompt 模板实现，在每个 turn 结束时自动注入。这标志着 coding agent 从单轮交互向持续自主执行的重要演进，与 Claude Code 的 /loop 和 Cursor 的 background agent 形成竞争态势。

simonwillison.net/2026/Apr/30/codex-goals/#atom... · Apr 30
蓝军
原文观点
这标志着 coding agent 从单轮交互向持续自主执行的重要演进。
蓝军视角分析
自主循环并非新概念——AutoGPT（2023）、Claude Code /loop、Cursor background agent 都已实现类似机制。Codex CLI 的 /goal 是跟进而非开创。更关键的是，摘要提到 agent「自评目标完成」作为终止条件，但自评可靠性是 agentic 系统的已知弱点：agent 判断自己「完成了」不等于目标真正达成。将 token 预算耗尽作为兜底机制，本质上承认了自评不可靠，这恰恰削弱了「重要演进」的叙事。
